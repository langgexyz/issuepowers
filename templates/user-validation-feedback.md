## 处理路径

| 类型 | 路径 |
|---|---|
| **小问题** | Agent 把本文件视为需求增量 → state 回到 `in-progress` → fix-forward-count += 1 → 重跑 plan → 实施 → merge → 部署 → 自动验证 → 再次请你验证 |
| **大问题** | 用 `/rollback <issue-id>`，把本次改动整个撤回 |

**强制约束**：fix-forward 已经修了 3 次还没通过验证，会强制要求回滚 —— 三次还修不好说明 understanding 本身就跟需求对不上，应该回滚之后重新 `/solve` 一遍。
